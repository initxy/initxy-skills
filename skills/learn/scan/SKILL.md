# Scan

## 铁律

1. **过滤，不总结。** 你的任务是帮用户决定值不值得花时间，不是帮他把内容读一遍。
2. **没具体相关性直接 drop。** "通用价值"、"值得了解"不算相关性，必须对应用户的具体项目、场景或已知兴趣。
3. **只给一个动作。** act / watch-1m / watch-3m / drop，不允许模糊建议或多选。

## 何时不用我

- 用户要理解内容本身（原理、概念、深度分析）— 转 `learn`。
- 内容是用户自己的项目代码或文档 — 不用我，直接看代码。
- 用户想把洞见固化为个人原则或项目约定 — 转 `sediment` 或 `capture`。

## 何时使用

- "看下这个链接 / release"
- "这周有什么重要的？"
- "这个我该关注吗？"
- "追一下最近变化"
- "有没有 breaking change / API 变更？"
- "值得现在试吗？"

## 行动语义

| 行动 | 含义 |
|---|---|
| **act** | 值得现在投入时间试用 / 接入 / 学习，窗口期短或收益明确。 |
| **watch-1m** | 方向对，但还不成熟或暂无场景，1 个月内回看。 |
| **watch-3m** | 潜力早期，3 个月内回看，不值得现在分心。 |
| **drop** | 无相关性、已知信息重复、或炒作大于实质，不再追。 |

## 流程

1. **识别真实变化。** 区分"新功能 / breaking change / 性能数据"与"营销文案、重复报道、泛泛评论"，后者直接丢弃。
2. **锚定相关性。** 相关性必须指向用户当前或近期的具体项目、技术栈或已知兴趣，不接受"通用价值"。
3. **给出唯一行动。** 依据相关性强度和时间窗口，选一个：act / watch-1m / watch-3m / drop。
4. **值得深读则交棒。** 若内容有长期学习价值，输出后注明"建议交 `learn` 深读"。
5. **极少数值得沉淀的交棒。** 独特视角或个人原则交 `sediment`，项目层约定交 `capture`。

## 输出

```markdown
## 变化
<1-3 句话，描述实际发生了什么>

## 重要性
<能力提升 / 风险 / 成本 / breaking change — 选最相关的维度>

## 相关性
<对应用户当前或未来的具体场景 — 必须具体，不能写"通用价值">

## 行动
act | watch-1m | watch-3m | drop
<一句话理由>
```

## 停止条件

- **来源打不开** — 要求用户提供内容原文，不猜测。
- **超过 3 条** — 分批处理，每批最多 3 条。
- **没具体相关性** — 直接输出 drop，不强行找关联。
- **值得深读** — 输出后转 `learn`，本 skill 不做深度分析。
