# 失败模式

填写 RFC 的失败行为和回滚预案时读取本文件。

## 外部依赖失败清单

每个外部依赖都要回答：

- 超时时系统怎么表现？
- 返回畸形数据时系统怎么表现？
- 变慢但没挂时系统怎么表现？
- 请求能否安全重试？
- 是否有熔断、队列缓冲、降级响应或硬错误？
- 用户可接受的可见行为是什么？

## 常见处理方式

| 失败 | 响应 |
|---|---|
| 只读依赖不可用 | 读缓存或返回明确降级响应 |
| 写依赖不可用 | 幂等时入队；非幂等时硬失败 |
| 第三方接口变慢 | 超时预算 + 熔断 |
| 队列积压 | 背压 + 告警 + 明确丢弃策略 |
| 迁移失败 | 停止发布 + 执行回滚迁移 |

## 回滚预案必须包含

- 触发条件。
- 明确回滚动作。
- 数据兼容性说明。
- 验证命令或可观察信号。
