# 技术栈来源

写 RFC 的「技术栈」小节时读取本文件。

## 优先级

1. **项目现有文件**：`AGENTS.md`、`CLAUDE.md`、`package.json`、lockfile、infra 文件、Docker 文件。
2. **已接受 ADR**：`docs/adr/` 下已生效的技术选型。
3. **个人默认栈**：`~/notes/principles/default-stack.md`，如果存在。
4. **一次性决策**：前三者都没有时，转 `decide`。

## 规则

- 项目现有栈优先于个人偏好。
- ADR 优先于从代码里推断出的惯例。
- 没有来源时，不要宣称「默认栈」。
- 偏离项目现有栈或 ADR 时，必须写证据和回滚代价。
