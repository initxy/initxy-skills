# 去 AI 味

中文 AI 味的四个来源:**翻译腔 + 模板化 + 正确的废话 + 不敢下判断**。逐一对治。写每一句都过这关,整篇写完再用末尾的速查清单扫一遍。

配套看 `techniques.md`:那份讲「该怎么写」,这份讲「别这么写」。

## 一、段末总结句(最高频,优先杀)

中文 AI 最爱在一段结尾再把刚说的复述一遍,拔个高。上一句说完就停。

要删的开头:「这说明……」「由此可见……」「由此我们可以看出……」「不难发现……」「到这里……」「综上……」「总的来说……」。

- ❌ 我们把重试加了退避。由此可见,合理的退避策略对系统稳定性至关重要。
- ✅ 我们把重试加了退避,雪崩没再出现过。

## 二、套话与华丽辞藻

说了等于没说的词,删;华丽但空的词,换成具体的。

| 不用 | 用 / 怎么办 |
|---|---|
| 众所周知 / 毋庸置疑 / 显而易见 / 不言而喻 | 删,直接说事实 |
| 值得注意的是 / 需要指出的是 | 删,直接说结论 |
| 随着……的发展 / 在当今……时代 | 删,直接开讲 |
| 综上所述 / 总而言之 / 总的来说 | 删,或换成一句具体收束 |
| 深刻 / 本质上 / 从某种意义上说 | 删,或讲清到底是什么 |
| 不可磨灭 / 范式转移 / 颠覆性 / 革命性 | 换成具体发生了什么 |
| 赋能 | 帮助 / 提供 |
| 抓手 | 关键措施 |
| 闭环 | 完整流程 |
| 沉淀 | 积累 / 形成 |
| 对齐 / 拉通 | 统一 / 打通 |
| 非常 / 极其 / 十分 | 很,或删 |
| 进行优化 / 做出调整 | 优化 / 调整 |

## 三、翻译腔

AI 中文的底色是英译中。这些句式是英语骨架套了中文词,要还原成中文的说法。

1. **抽象名词当主语** → 让人或具体的东西当主语。
   - ❌ 缓存的引入带来了一致性的挑战。
   - ✅ 加了缓存,数据就容易和数据库对不上。

2. **滥用「被」字** → 改主动或无主语。
   - ❌ 这个 bug 被我在日志里发现了。
   - ✅ 我在日志里翻到了这个 bug。

3. **虚化动词堆砌**(进行 / 作出 / 给予 / 加以 / 予以 / 实现)→ 用实义动词。
   - ❌ 对数据进行清洗,对结果作出验证。
   - ✅ 清洗数据,验证结果。

4. **长定语前置** → 拆成短句,或把修饰后置。
   - ❌ 这是一个用于解决高并发场景下缓存与数据库数据不一致问题的方案。
   - ✅ 这个方案解决一个问题:高并发下缓存和数据库对不上。

5. **「和」连接非名词** → 改「并」、顿号,或拆句。
   - ❌ 我们需要监控和优化这些慢查询。
   - ✅ 这些慢查询要监控,也要优化。

6. **滥用数量词**(一个 / 一种 / 一台)→ 能删就删。
   - ❌ 这是一个很好的一个设计。
   - ✅ 这个设计很好。

7. **冗余时态词**(已经 / 曾经 / 正在,中文靠上下文)→ 删。
   - ❌ 我们已经在去年就已经完成了迁移。
   - ✅ 迁移去年就做完了。

8. **「由于……导致……」生硬搭配** → 拆成两句,让因果自然。
   - ❌ 由于连接没有释放,导致连接池被耗尽。
   - ✅ 连接没释放,连接池很快被占满。

9. **句子过长** → 一句塞太多,切短。一个逗号串四五个分句就是信号。

## 四、模板化结构

- **机械的「首先 / 其次 / 再次 / 最后」** → 融成有因果和递进的段落,靠句意本身过渡,不靠序号。需要并列时用列表,不用「首先其次」凑。
- **「N 大秘诀 / 五个要点 / 全面解析」式标题党** → 换成具体说了什么的标题。
- **总分总八股**(开头「本文将从多个维度深入探讨」,结尾「希望本文对你有帮助」)→ 都删。
- **三连排比连发**:一组排比有力,背靠背连用三四组就是模式化,换节奏。

## 五、不敢下判断

该断言时别含糊。这些是回避判断的信号词,出现就问自己:我到底什么观点?

「在一定程度上」「某种意义上」「可能也许」「需要根据具体情况」「见仁见智」「因人而异」。

有真观点就说死(见 `techniques.md` 第 4 条);真不确定就老实说不确定(第 8 条)。最差的是用模糊词假装稳妥。

## 六、连接词过载

AI 平均每句都挂个连接词:然而、因此、此外、与此同时、不仅……而且、值得一提的是。中文容许句子之间留白,靠语义接续。删掉一半连接词,通常更顺。

- ❌ 然而,我们发现了问题。因此,我们做了优化。此外,我们还加了监控。
- ✅ 我们发现了问题,做了优化,顺手加了监控。

## 标点与排版

- 简体中文标点;引号用直角引号「」。
- 中文与英文 / 数字之间留一个空格(版本 2.0、用 Docker 跑)。
- 不用破折号 ——,改逗号或句号。
- 代码、JSON 键名、URL、路径原样不动。

## 正反例

> ❌ 众所周知,随着云原生的不断发展,Kubernetes 已成为容器编排的事实标准。值得注意的是,它并不是银弹,本文将从多个维度深入探讨其落地挑战。

> ✅ 我们用 Kubernetes 跑了三年生产。它解决了部署,却把复杂度搬到了别处。这篇记录我们踩的五个坑,和各自的止损办法。

> ❌ 通过对系统进行全面的优化,我们成功地实现了性能的大幅提升,这充分说明了优化工作的重要性。

> ✅ 改了三个地方:连接池、索引、批量写。P99 从 800 毫秒掉到 90。
