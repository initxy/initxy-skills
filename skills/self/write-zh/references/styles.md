# 语言风格

`SKILL.md` 里的默认风格是「第一性为主 + 传统文化淡点缀」,那是这套写作的底色。这份给几种可切换、可借鉴的风格画像。

用法:按文章类型挑一种为主,可以和默认底色混。整篇保持一种主风格,别一段科普一段硬核地跳。每种风格各有招牌动作,也各有翻车点。

## 怎么选

| 你要写的 | 主风格 |
|---|---|
| 把一个概念 / 原理讲清楚 | 清淡科普 |
| 经验总结、架构观点、踩坑复盘 | 老练实战 |
| 有强观点,要从头讲清「为什么」 | 第一性硬核 |
| 上手教程、轻松向、个人随笔 | 口语个人 |
| 团队博客、设计文档、对外正式输出 | 严谨工程 |

---

## 清淡科普

**适合**:讲清一个概念、协议、算法、原理。目标是让外行也读懂。

**声音**:像在整理自己的学习笔记,平静、克制、不炫技。第一人称,坦白「我也是刚搞懂」。深入浅出靠的是顺序,不是简化。

**招牌动作**:
- 开头交代为什么写这篇(常是「我想搞清楚 X」)。
- 从历史或最朴素的问题切入,一步步往上推。
- 句子短,段落短,一段两三句。
- 抽象概念立刻配一个具体例子(「以汉字『严』为例」)。
- 编号小节当骨架,结尾干净利落,不拖。

**翻车点**:别为了「浅」而牺牲准确;别堆细节淹没主线。清淡不是简陋,是把复杂的东西排好序。

## 老练实战

**适合**:经验总结、架构原则、技术选型、踩坑复盘。

**声音**:干了很多年、见过很多场面的人在跟同行聊。观点鲜明,甚至带点火气。不端着,和一线程序员站在一起。

**招牌动作**:
- 用经历开场建立底气(「这些年我见过太多……」)。
- 观点直接抛出,再用真实案例和反例撑。反例往往比正例更有说服力。
- 长短句搭配:长句讲清来龙去脉,短句砸结论(「这是失败的。」)。
- 接地气的比喻(「在歪楼上接楼」)。
- 把经验提炼成「原则一、原则二」这样的体系。

**翻车点**:火气可以有,但别变成单纯吐槽。每个判断后面要有事实,不然就成了情绪。

## 第一性硬核

**适合**:有强观点、要挑战常见说法、从基本原理把一件事讲到底的深度文。

**声音**:从定义和基本事实出发,自己推,不靠「业界一般认为」。断言式,用词简洁有力,几乎不寒暄。

**招牌动作**:
- 先把概念还原到最朴素的形式,再往上推结论。
- 敢反驳流行说法,但反驳靠论证,不靠扣帽子。
- 句子干净,一句一个意思,不留废话。
- 结论下得斩钉截铁(「真正优雅的代码,几乎不需要注释」)。

**翻车点**:这种风格最容易滑向刻薄、攻击人。**只取它的论证方式和简洁,不取攻击性**。批评观点,不贬低人。断言要担得起,担不起就降级成「我倾向于认为」。

## 口语个人

**适合**:上手教程、轻松科普、个人随笔。读者是来学一个具体东西的。

**声音**:像朋友手把手带你,极口语,有生活气,会自嘲。把枯燥的东西讲得轻松。

**招牌动作**:
- 大白话直说,不绕(「记住 justify 这个词,justify-content 也就记住了」)。
- 拟人、打比方让概念有画面(把浏览器、元素当成会说话的角色)。
- 坦诚「这块我也是边学边写」,亲和。
- 适当用语气词、感叹号、谐音梗,但别过量。
- 结尾轻:一句「感谢阅读,有错欢迎指正」就够。

**翻车点**:口语不等于啰嗦。该精确的地方(参数、行为、边界)还得精确,别用「大概」「差不多」糊过去。

## 严谨工程

**适合**:团队技术博客、设计文档、对外正式输出。多人会读,要经得起推敲。

**声音**:客观、克制、中立。用「我们」。给数据、给权衡,既说优点也说缺点。

**招牌动作**:
- 结论给限定条件,不夸大(「在我们这个量级下……」)。
- 关键判断配数据或公式,不靠形容词。
- 优缺点成对出现,主动暴露代价。
- 结构层级清晰,图表 / 架构图驱动。

**翻车点**:客观克制最容易滑回 AI 味,满篇「在一定程度上」「需要综合考虑」。克制是少用情绪词,不是不下结论。该给的判断和数字一个都不能少,照样过一遍 `de-ai.md`。
