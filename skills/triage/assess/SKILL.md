# Assess

## 铁律

1. 不为低价值工作做详细方案。结论是 do-not 就停，不给安慰性路线图。
2. 条件影响结论时不给无条件 go。前置条件不满足，结论必须是 do-later 或 do-not。
3. 弱证据必须标注。假设未经验证时强制在对应字段加 `evidence: weak`。

## 何时不用我

- 30 分钟内可完成且可逆的事：直接做，不进流程。
- 用户已有强决心、只是要情感支持或执行陪伴：你不是在帮他决策，转到 `plan` 或 `tdd`。
- 已明确是 A vs B 选项对比：转 `decide`，那是比较框架，不是立项框架。

## 何时使用

- "这个方向值不值得做？"
- "现在要不要启动这个项目 / 功能？"
- "优先级高吗？还是先做别的？"
- "引入这个框架 / 架构 / 技术债值不值？"
- "时机对不对？为什么是现在？"
- "这个需求是真需求还是假需求？"

## 模式

默认由场景决定：产品方向 / 新业务 / 立项早期 → **yc**；功能内决策 / 工程改进 / 技术债 → **roi**。用户也可显式指定 `mode: yc` 或 `mode: roi`。

### mode: yc（YC 立项框架，借鉴 Paul Graham 5 问扩展版）

逐一回答 8 个问题，缺答案时标 `evidence: weak`：

| # | 问题 | 说明 |
|---|---|---|
| 1 | customer | 具体是谁？不接受"所有人"或"企业" |
| 2 | pain | 急性痛（acute）还是慢性痛（chronic）？ |
| 3 | timing | 为什么是现在？市场/技术/监管发生了什么？ |
| 4 | edge | 为什么是你？不公平优势是什么？ |
| 5 | distribution | 怎么触达他们？第一批用户从哪来？ |
| 6 | scale | 10x 之后长什么样？规模图景清晰吗？ |
| 7 | insight | 别人都错过了什么洞察？ |
| 8 | killer | 最可能致命的失败模式是什么？ |

### mode: roi（工程内决策框架，借鉴 superpower 验证闸门）

4 个维度，每个维度必须量化或给范围，不接受"大概"：

| 维度 | 要回答的 |
|---|---|
| 价值 | 为谁创造多少价值？有没有数据或代理指标？ |
| 成本 | 开发 + 维护 + 迁移 + 认知负担，总计多少？ |
| 风险 | 技术风险 / 依赖风险 / 回滚难度？ |
| 可逆性 | 做了能不能撤？撤的代价？ |

## 流程

1. 判断 mode（yc 或 roi），不确定时问用户一句话：产品方向还是工程内决策？
2. 按 mode 逐项填写框架；信息不足的字段打 `evidence: weak`，不猜测。
3. 检查是否有更简单替代方案（是否存在一个更小的 MVP 或更可逆的方案能先验证核心假设）。
4. 给出唯一结论：do-now / do-different / do-later / do-not。
5. **完成后强烈推荐走 `grill`**：用对抗视角检验结论是否需要降级。

```markdown

- edge: <不公平优势> [evidence: weak?]
- distribution: <触达路径>
- scale: <10x 图景>
- insight: <别人错过的洞察>
- killer: <最可能的致命失败模式>

## 结论
do-now | do-different | do-later | do-not

## 理由（一句话）
<最关键的支撑或阻碍因素>

## 前置条件（如有）
- <做之前必须先验证什么>

## 下一步

```

### roi 模式模板

```markdown
## assess (mode: roi)

- 价值: <为谁 / 多少> [evidence: weak?]
- 成本: <开发 X 天 / 维护 Y / 迁移 Z / 认知负担?>
- 风险: <技术/依赖/回滚>
- 可逆性: <可逆 / 部分可逆 / 不可逆，撤销成本?>

## 结论
do-now | do-different | do-later | do-not

## 理由（一句话）
<最关键的支撑或阻碍因素>

## 前置条件（如有）
- <做之前必须先满足什么>

## 下一步
clarify | design | grill | drop
```

## 停止条件

- 说不清谁受益、什么会变好 → 转 `clarify`，先把问题定义清楚。
- 已明确是 A vs B 选项对比 → 转 `decide`。
- 用户想让结论被攻击、找漏洞 → 转 `grill`。
