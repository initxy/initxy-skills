# 对齐追问模式

运行 `grill --align` 时读取本文件。

## 好问题

- 「如果只能保留一个用户场景，是哪个？为什么？」
- 「谁会最早感受到这个问题？他们今天怎么绕过去？」
- 「做完三个月后，哪个可观察信号证明它有效？」
- 「哪个功能最容易被砍掉而不伤核心目标？」
- 「这个方案最可能在哪个边界条件下失败？」
- 「你说的 X 是业务概念，还是实现细节？」

## 输出格式

```markdown
## 当前理解
<一句话>

## 问题
<一个具体问题>

## 我的推荐
<推荐答案 + 一句理由>
```

## 停止标准

- 目标具体。
- 用户/场景具体。
- 范围和非目标明确。
- 验收可观察。
- 最大风险已命名。
