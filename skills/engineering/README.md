# Engineering

日常工程用 skill。

- **[diagnose](./diagnose/SKILL.md)** — 结构化诊断 bug 与性能回归。
- **[grill-with-docs](./grill-with-docs/SKILL.md)** — 结合项目语言和 ADR 追问方案，并即时更新文档。
- **[triage](./triage/SKILL.md)** — 通过状态机整理 issue。
- **[improve-codebase-architecture](./improve-codebase-architecture/SKILL.md)** — 寻找 deepening 机会，提升可测试性和可导航性。
- **[product-shaping](./product-shaping/SKILL.md)** — 将模糊产品想法收敛成 PRD 前置 Product Brief。
- **[setup-matt-pocock-skills](./setup-matt-pocock-skills/SKILL.md)** — 初始化 issue tracker、triage 标签和领域文档配置。
- **[ship](./ship/SKILL.md)** — 检查、验证、commit、push，并在有明确流程时 publish。
- **[tdd](./tdd/SKILL.md)** — 红绿重构式测试驱动开发。
- **[to-issues](./to-issues/SKILL.md)** — 把计划拆成可独立领取的垂直切片 issue。
- **[to-prd](./to-prd/SKILL.md)** — 从当前上下文生成 PRD 并发布到 issue tracker。
- **[to-sdd](./to-sdd/SKILL.md)** — 在 PRD 后、拆 issue 前生成软件设计文档。
- **[zoom-out](./zoom-out/SKILL.md)** — 上升一层理解代码区与整体系统关系。
- **[prototype](./prototype/SKILL.md)** — 构建一次性逻辑或 UI 原型。
- **[review](./review/SKILL.md)** — 从固定点 review diff：Standards 与 Spec 两条轴并行。
