# Deep Modules

来自《A Philosophy of Software Design》：

深模块 = 小接口 + 厚实现。

```text
┌─────────────────────┐
│   Small Interface   │  少方法、简单参数
├─────────────────────┤
│                     │
│  Deep Implementation│  复杂逻辑被隐藏
│                     │
└─────────────────────┘
```

浅模块 = 大接口 + 薄实现。避免。

```text
┌─────────────────────────────────┐
│       Large Interface           │  方法多、参数复杂
├─────────────────────────────────┤
│  Thin Implementation            │  基本只是转发
└─────────────────────────────────┘
```

设计接口时问：

- 方法数能不能更少？
- 参数能不能更简单？
- 能不能把更多复杂度藏进内部？
