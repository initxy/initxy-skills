# 发布检查清单

准备非平凡拉取请求（PR）、发布或部署时读取本文件。

## 必查项

- 工作区已检查；无关未提交文件已列出。
- 当前分支相对上游/基准分支的状态已检查。
- 验证命令已新鲜运行。
- diff review 已完成。
- changelog 按用户影响写好。
- 回滚预案已写。

## 阻塞项

- 验证失败。
- review 有 P0/P1 finding。
- 破坏性变更没有明确回滚方案。
- 分支明显落后基准分支。
- 会把无关未提交文件带进发布。

## 必须用户确认的动作

- push。
- 强制推送。
- 创建发布版本。
- 部署。
- 对共享环境执行迁移。
