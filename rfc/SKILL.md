# RFC

## 铁律

1. **没有 NFR 的设计不是 RFC，是草图。** NFR 必须有数字：响应时间用 ms，可用性用 9 的数量，吞吐用 rps/qps，不接受"快"、"高可用"、"可靠"等形容词。
2. **没有失败模式与回滚预案的方案不进入实现。** 每个外部依赖必须描述它挂掉时系统的行为；每次发布必须有可执行的回滚步骤。
3. **跨边界处必须有契约。** 凡跨模块、跨服务、跨团队的调用，必须定义 API 请求/响应、错误码、幂等性，或事件/消息格式。
4. **默认有埋点。** log / metric / trace 接入点必须在可观测性 section 列出，不能上线后补。
5. **RFC 不被 `grill` 不算定稿。** 写完草稿后必须强制走 `grill`；skeptic 没有找不到漏洞的 RFC，只有没被攻击过的 RFC。
6. **rfc 不直接写 ADR**——重大决策由 `capture` 写独立 ADR 文件，本 RFC 内用链接引用。

## 何时不用我

- **单模块、小功能、私下探索**：30 分钟可逆的决策 → 走 `design`。
- **改存量代码结构**：不新增对外接口、不影响其他模块 → 走 `refactor`。
- **还不确定值不值得做** → 先 `assess`，再 `clarify`，再来 RFC。
- **技术选型还没定** → 先 `decide` 锁选型，再写 RFC。

## 何时使用

- 方案跨 3+ 模块，或跨团队/服务边界。
- 有 SLA、容量或性能目标需要写进合同或 OKR。
- 涉及对外 API 契约（REST / gRPC / GraphQL / 事件 schema），变更影响下游。
- 预期维护周期 > 3 个月，或要交接给他人。
- 需要多人 review 或存档决策理由（ADR 挂载点）。
- 涉及合规（GDPR / SOC2 / 金融监管）或数据安全分级。
- 涉及钱：infra 成本、API 调用费、人力。

## 流程

**1. 读上下文**
读 `CONTEXT.md`、`docs/adr/`、`CLAUDE.md`；不存在则跳过。

**2. 提取目标**
从 `clarify` 输出或用户描述：可观察目标 / 明确非目标 / 为什么是现在。

**3. 确定技术栈**
先列默认栈，再标偏差及证据。偏差无证据 = 退回默认栈。

**4. 按接缝拆模块**
接缝 = 数据所有权边界 + 变更独立性边界。每个接缝处必有契约。

**5. 逐一写 NFR**
每条 NFR 必有数字 + 测量方式。例：「P95 < 200ms（k6，100 并发，5 分钟）」。

**6. 枚举外部依赖与失败行为**
每个依赖（DB / 队列 / 第三方 API / 内部服务）写 fail 时的降级：熔断 / 队列缓冲 / 降级响应 / 硬错误。

**7. 写可观测性规划**
每个模块的关键 log（级别 + 字段）、metric（名 + 标签 + 聚合）、trace span 名称。

**8. 写回滚预案**
如何撤销发布：feature flag / 版本回退 / 数据迁移回滚 / 客户端兼容期。

**9. 强制 `grill`**
草稿完成后调用 `grill` 红队攻击。attack points 补回 RFC 或在「待 grill 的部分」标注。**未经 grill 的 RFC 不算定稿。**

**10. 大决策点输出 ADR（由 `capture` 写）**
重大技术选型（存储引擎、通信协议、一致性模型）由 `capture` 写独立 ADR，本 RFC 用链接引用。**rfc 不直接写 ADR**，保持职责分离。

**存储位置**：`docs/rfc/<YYYY-MM-DD>-<slug>.md`。状态变更原地更新文件头部字段，不新建。

## 输出

**完整 RFC 模板**：见 [`rfc/templates/full-rfc.md`](templates/full-rfc.md)，包含每个 section 的字段示例和示意值。

**骨架**（每个 section 都不能缺，详细字段去模板看）：

```markdown
# RFC-<编号>: <标题>
**作者** / **日期** / **状态** (draft/review/accepted/superseded) / **相关 ADR**

## 摘要 / 目标 / 非目标
## 上下文与动机
## 技术栈（含默认栈偏差表）
## 模块清单
## API 契约（每个对外接口：端点 / 请求 / 响应 / 错误码 / 幂等性）
## 数据模型（表 / 字段 / 索引 / 约束 / 迁移策略）
## 数据流（端到端，可含序列图）
## 状态机（如有有状态实体）
## NFR（每条带数字 + 测量方式）
## 失败模式与降级（每个外部依赖）
## 可观测性（logs / metrics / traces）
## 安全模型（认证 / 授权 / 数据敏感度 / 威胁场景）
## 成本估算（infra + 人力）
## 风险与 Trade-off
## 开放决策
## 回滚预案
## 里程碑
## 待 grill 的部分
```

## 停止条件

- 目标仍模糊 → 转 `clarify`。
- 技术选型未定 → 转 `decide`。
- 实际是小改动 → 转 `design`。
- 实际是存量结构改造 → 转 `refactor`。
- 假设未经验证 → 转 `assess`。
- 草稿完成 → 强制走 `grill`；未 grill 不算定稿。
